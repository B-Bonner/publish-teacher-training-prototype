We created the logic around course folding in a design with data exercise that we have since tested with training providers and candidates:

  • Training provider research
  • Candidate research

The first version of the tool will be based on UCAS’s definition of a course. Course metadata will still be created and configured using UCAS. The UCAS model of a course must persist in some format in the publish courses tool. It is also the model that providers are most familiar with.

Folded courses didn’t test well

When we tested folding courses by subject with training providers they expressed surprise and confusion. It wasn’t obvious to them how certain information would be explained in the fields provided. For example: ‘About this course’, “which course is that for, these courses are different?”

We iterated the page layout to be more explicit about the fields for each course variant to avoid this confusion. We found more and more small exceptions that meant fewer fields could be shared between variants; eg a salaried course has a different interview process, and might have more emphasis on experience in the entry requirements.

When we tested with candidates it wasn’t clear that a page represented different variants of a course, and it wasn't obvious which fields related to which variant.

Folding on less

In response to research and to reduce complexity for users, we will fold only in the scenario where a school direct provider has created multiple courses for each of their schools.

Where schools have been given their own subject course by their training provider, these can be combined into a single course – as though they were one couse on UCAS with multiple campus options.

Unfolding full time and part time

Training providers can choose to represent full time and part time courses as a single course in UCAS. Part time courses can be longer and have different placement school policies. We need to unfold them and ask for information for the part time version.

In this scenario both full time and part time would have the same course code, the latter would be denoted by selecting "part time" in the application process.

Some providers already separate full time and part time courses.
